Intensity Transform Functions

85% · Part A/B

What is Image Enhancement?

Image Enhancement improves the visual appearance of an image or makes features more perceptible for human viewers or automated processing. Unlike restoration (Unit 4), enhancement is subjective — there is no single "correct" result.

Enhancement approaches: (1) Spatial domain — operate directly on pixels f(x,y). (2) Frequency domain — operate on Fourier transform F(u,v).

General form of spatial enhancement: g(x,y) = T[f(x,y)] where T is the transformation and f, g are the input/output images.

⭐ 1. Image Negatives (Linear)

Reverses the intensity levels. Useful for displaying bright details in dark areas of an image (like X-rays).

Image Negative
s = (L - 1) - r
r = input pixel intensity, s = output intensity, L = number of gray levels (256 for 8-bit)
For 8-bit: s = 255 - r. White→Black, Black→White.

⭐⭐ 2. Log Transformation

Compresses the dynamic range of pixel values. Useful when input data spans a very wide range (like Fourier spectrum which has one very bright DC component and many small values).

Log Transformation
s = c · log(1 + r)
c = scaling constant (chosen so output fits in [0, L-1]). r ≥ 0.
log(1+r) not log(r) → avoids log(0) when r=0.
Effect: Dark regions are brightened (compressed range). Bright areas appear similar.
💡
When to Use Log Transform — Exam Trick

Log transform is ideal for displaying the Fourier spectrum. The spectrum has values from 1 to 10⁶+, which cannot be displayed on a normal 8-bit screen. Log compress this to about 0–14. In the exam: "Why use log transform to display Fourier spectrum?" → "To compress the huge dynamic range so low-value components are also visible."

⭐⭐⭐ 3. Power-Law (Gamma) Transformation — MOST ASKED

Also called gamma correction. The parameter γ controls the shape of the mapping curve. Very widely used in display systems, cameras, and printing.

Power-Law / Gamma Transformation
s = c · rᵞ
c = constant (usually 1). γ = gamma value.
γ < 1 → maps dark input to brighter output (brightens image)
γ > 1 → maps bright input to darker output (darkens image)
γ = 1 → linear (identity, no change)
Fig 1.1 — Power-law transform: γ<1 brightens, γ>1 darkens. γ=1 is identity.

⭐⭐ 4. Piecewise-Linear Transformations

a) Contrast Stretching

Expands the range of intensity values to span the full [0, L-1] range. Improves contrast of low-contrast images (where all pixels cluster in a narrow intensity range).

Contrast Stretching Formula
s = (r - r_min) / (r_max - r_min) × (L-1)
r_min, r_max = actual min and max in input image. Output fills [0, 255].
More general: map specific range [r1,r2] to [s1,s2] using three line segments.

b) Intensity Slicing (Gray-Level Slicing)

Highlights a specific range of intensities. Two variants:

  • With background: Highlight [A,B] to bright value; rest stays as original
  • Without background: Highlight [A,B] to bright value; rest → 0 (black)

Use: Highlighting specific structures in medical images (e.g., blood vessels in a specific density range).

c) Bit-Plane Slicing

Decomposes an 8-bit image into 8 binary planes. Bit plane 7 (MSB) contains the most significant structure; bit plane 0 (LSB) contains the least.

Bit Plane Extraction
Bit plane k = floor(f(x,y) / 2ᵏ) mod 2
k = 0 (LSB) to 7 (MSB). Higher planes carry more image information.
Image compression: transmit/store only the top 4–5 bit planes with minimal quality loss.

Bit plane 7 (MSB): 1 (most significant) | Bit plane 0 (LSB): 1

Contrast Stretching:
Contrast stretching expands the range of intensity values in a low-contrast image to span the full intensity range [0, 255]. It is a piecewise-linear transformation.
Formula: s = (r − r_min)/(r_max − r_min) × 255
Example: Image with intensities only in [50, 150] is stretched to [0, 255]. Every 50→0, 150→255.
The transformation function is an S-shaped (three-segment) piecewise linear function.

Intensity Slicing (Gray-Level Slicing):
Highlights a specific band of gray levels [A, B] to a bright value (e.g., white = 255), while other intensities are either left unchanged (with-background) or set to black (without-background).
Use case: In MRI, highlighting pixels in a specific intensity range corresponding to bone vs soft tissue.
Two modes:
• Binary: output = 255 if A ≤ r ≤ B, else 0
• With background: output = 255 if A ≤ r ≤ B, else r (unchanged)

Intensity Transform Functions:
Intensity transforms operate on single pixels: s = T(r), where r = input intensity, s = output intensity.

1. Image Negatives: s = 255 − r. Graph: straight line with negative slope from (0,255) to (255,0). Used for: X-ray analysis, photographic negatives.

2. Log Transformation: s = c·log(1+r). Graph: steep rise at low r, flattens at high r (concave down). Compresses dynamic range. Used for: displaying Fourier spectrum.

3. Power-Law (Gamma): s = c·rᵞ.
• Graph for γ<1: concave (above diagonal) → brightens dark pixels
• Graph for γ>1: convex (below diagonal) → darkens bright pixels
Used for: gamma correction in displays, cameras, printing.

4. Piecewise-Linear (Contrast Stretching): Three-segment line. Maps [r1,r2] to [s1,s2] with (0→0) and (255→255) endpoints. Graph: S-shaped for typical contrast enhancement.

5. Threshold Function: s = 0 if r<T, s = 255 if r≥T. Graph: step function. Produces binary image.

Histogram Processing

🔥 90% · Part B/C Numerical

What is a Histogram?

The histogram of a digital image with gray levels [0, L-1] is a discrete function h(rₖ) = nₖ, where rₖ is the kth gray level and nₖ is the number of pixels with that gray level.

Normalized Histogram (Probability Estimate)
p(rₖ) = nₖ / n for k = 0, 1, ..., L-1
n = total number of pixels = M×N. p(rₖ) ≈ probability that a pixel has gray level rₖ.

A dark image has histogram concentrated in low values. A bright image has histogram concentrated in high values. A high-contrast image has histogram spread evenly.

⭐⭐⭐ Histogram Equalization — Full Algorithm

Histogram equalization redistributes gray levels to produce a nearly uniform histogram — spreading intensities evenly across [0, L-1]. This automatically improves contrast.

Histogram Equalization Transformation
s_k = T(r_k) = (L-1) · Σᵢ₌₀ᵏ p(rᵢ) = (L-1) · CDF(r_k)
s_k = output gray level for input r_k. CDF = Cumulative Distribution Function.
Practical discrete formula: s_k = round[ ((L-1)/n) · Σᵢ₌₀ᵏ nᵢ ]

✅
Self-Check for Histogram Equalization

1. Sum of all nₖ = total pixels (M×N) ✓
2. Last CDF value = n = M×N ✓ (so last output = L-1 = 255)
3. Mapping is monotonically non-decreasing ✓ (higher r_k → higher or equal s_k)
4. All output values lie in [0, L-1] ✓
5. Count the unique output values — fewer than input → some gray levels are merged (normal!)

⭐⭐ Histogram Matching (Specification)

Instead of equalizing to a uniform histogram, match the image histogram to a specified target histogram. More flexible than equalization — can target any desired distribution.

1
Equalize the input image using histogram equalization → get mapping r → s.
2
Equalize the target histogram (desired distribution) → get mapping z → v.
3
For each s value, find the z that maps to the closest v = s → that z is the output.
4
Compose: r → s → z gives the complete histogram specification mapping.
🔗
Connection: Histogram → Unit 5 (Compression)

The histogram of an image is directly used in Unit 5 compression! p(rₖ) from the histogram = probability of each gray level = input to Huffman coding. High-frequency gray levels get shorter codes. This is exactly why compression works better on images with non-uniform histograms.

Spatial Filtering

🔥 90% · Numerical Heavy

What is Spatial Filtering?

Spatial filtering modifies pixel values based on the values of surrounding pixels using a filter/mask/kernel. The filter is slid over the image and at each position, a new pixel value is computed from the neighborhood.

Two types: (1) Smoothing (low-pass) — reduces noise, blurs image. (2) Sharpening (high-pass) — enhances edges, increases contrast.

⭐⭐⭐ Correlation vs. Convolution — MUST KNOW DIFFERENCE

Correlation — Slides Filter WITHOUT Flipping
(f ⊛ w)(x,y) = Σₛ Σₜ w(s,t) · f(x+s, y+t)
Filter is placed at (x,y) and slides. No flipping of kernel.
Used in: template matching, cross-correlation.
Convolution — Slides Filter WITH 180° Rotation
(f ★ w)(x,y) = Σₛ Σₜ w(s,t) · f(x-s, y-t)
Filter is rotated 180° before sliding. This is "true" linear filtering.
For symmetric kernels (Gaussian, Box): convolution = correlation!
🧠
Correlation vs Convolution — Exam Trick

Correlation: Flip NOTHING. Just slide the kernel and multiply.
Convolution: Rotate kernel 180° (flip both horizontally and vertically), THEN slide.
For any symmetric kernel (same when rotated): correlation = convolution.
The 2023 and 2024 PYQ (Part C Q3) uses both — filter F is Gaussian (symmetric = same), filter L is Laplacian (asymmetric → flip for convolution).

Box vs Gaussian — RTU Short Answer

Box filter: all weights equal = 1/9. Simple, fast. Creates "blocky" blur — ringing artifacts at edges.
Gaussian filter: center weighted more (1,2,4,2,1 pattern). Smoother blur. Isotropic. No ringing.
Key difference (2023 Part A Q3): Box filter has flat frequency response causing ringing (Gibbs effect); Gaussian has smooth Gaussian frequency response (no ringing).

Sharpening (High-Pass) Spatial Filters

1. First Derivative — Gradient (Sobel, Prewitt)

Computes gradient magnitude → highlights edges. Already covered in Unit 6 (segmentation). Sobel Gx, Gy kernels detect vertical and horizontal edges respectively.

2. Second Derivative — Laplacian

g(x,y) = f(x,y) + ∇²f(x,y) [if center of mask is positive]
Subtracting negative-center Laplacian = adding sharpening. Rule: match the sign to center coefficient.

3. Unsharp Masking

A popular technique in photography and printing. Subtracts a blurred version from the original to get the "unsharp mask" (the edges), then adds it back.

Unsharp Masking Process
1. Blur: f_blur = f ★ Gaussian
2. Mask: f_mask = f - f_blur (this IS the high-frequency content)
3. Sharpen: g = f + k·f_mask
k = amount of sharpening. k=1 → standard unsharp masking. k>1 → highboost filtering.
💡
Unsharp Masking vs Laplacian Sharpening — 2 marks Distinction

Laplacian sharpening: Uses second-derivative operator directly. Fast. Single kernel application.
Unsharp masking: Subtracts blurred version. More control (adjust k). Traditional darkroom technique now done digitally. Equivalent to high-boost filtering.
Both achieve edge enhancement, but unsharp masking gives more control over the amount of sharpening via the k parameter.

Spatial Filtering:
Spatial filtering modifies pixel values based on their neighborhood using a filter kernel w(s,t). The kernel is moved over the image and at each position a new value is computed. Used for smoothing (noise removal) and sharpening (edge enhancement).

Spatial Correlation:
(f ⊛ w)(x,y) = Σₛ Σₜ w(s,t) · f(x+s, y+t)
The kernel w is slid over the image WITHOUT any flipping. The kernel acts as a template — correlation measures how similar the kernel is to the local image region.

Spatial Convolution:
(f ★ w)(x,y) = Σₛ Σₜ w(s,t) · f(x-s, y-t)
The kernel w is first rotated by 180° (flipped both horizontally and vertically), then slid over the image.

Key Difference:
CorrelationConvolution
No kernel flipKernel rotated 180°
Template matchingLinear filtering (mathematical)
Not commutative in generalCommutative: f★w = w★f
DFT: F* · W (complex conjugate)DFT: F · W (direct multiplication)
Used for: feature detectionUsed for: image filtering, convolution theorem

When are they equal? When the kernel is symmetric (same after 180° rotation) — e.g., Gaussian, box filter, Laplacian. NOT equal for asymmetric kernels like Sobel Gx.

Fourier Transform & Properties

85% · Part A/B

1D Discrete Fourier Transform

1D DFT (Forward)
F(u) = Σₓ₌₀^(N-1) f(x) · e^(-j2πux/N) for u = 0,1,...,N-1
1D IDFT (Inverse)
f(x) = (1/N) · Σᵤ₌₀^(N-1) F(u) · e^(j2πux/N) for x = 0,1,...,N-1

2D Discrete Fourier Transform (Most Important)

2D DFT
F(u,v) = Σₓ Σᵧ f(x,y) · e^(-j2π(ux/M + vy/N))
2D IDFT
f(x,y) = (1/MN) · Σᵤ Σᵥ F(u,v) · e^(j2π(ux/M + vy/N))
M×N = image size. F(u,v) = complex frequency spectrum. F(0,0) = DC component = MN × mean intensity.

⭐ Key Properties of 2D DFT

PropertyMathematical StatementPractical Meaning
Linearity DFT{af+bg} = a·DFT{f} + b·DFT{g} DFT of weighted sum = weighted sum of DFTs
Translation (Shift) f(x-x₀,y-y₀) ↔ F(u,v)·e^(-j2π(ux₀/M+vy₀/N)) Shifting image = multiplying spectrum by phase. Only phase changes, magnitude stays same.
Convolution Theorem f★h ↔ F(u,v)·H(u,v) Convolution in spatial = multiplication in frequency. This is WHY frequency domain filtering is fast!
Correlation Theorem f⊛h ↔ F*(u,v)·H(u,v) Correlation = multiply by complex conjugate in frequency domain
Periodicity F(u,v) = F(u+M,v) = F(u,v+N) DFT spectrum is periodic → causes wraparound error if not handled
Symmetry F(u,v) = F*(-u,-v) For real images, spectrum is conjugate symmetric. |F(u,v)| = |F(-u,-v)|
Rotation Rotating f by θ → rotating F by θ Rotation in spatial = rotation in frequency domain
🧠
Most Exam-Critical Properties — Convolution + Shift

Convolution theorem (f★h ↔ F·H): This is THE most important. It says: instead of slow spatial convolution (O(N²M²)), take two FFTs, multiply pointwise, then IFFT → much faster (O(NM·log(NM))).
Shift property: Moving an image doesn't change the magnitude spectrum — only the phase. So |F| (power spectrum) is shift-invariant. The 2023 Part A Q2 asks exactly this!

Centering the Spectrum

By default, DFT(0,0) — the DC component — appears at the corner of the spectrum. To center it (DC at the center), multiply the image by (-1)^(x+y) before taking DFT. This is called "centering" or "fftshift" in MATLAB/Python.

f(x,y)·(-1)^(x+y) ↔ F(u-M/2, v-N/2)
This shifts the DC component from corner (0,0) to center (M/2, N/2). Used for displaying the spectrum.
📝 RTU PYQs — Fourier Transform
Q. Explain shifting and convolution properties of 2-D discrete Fourier transform.
June 2023 Part A Q2 · 2 marks
Shifting Property of 2D DFT:
If f(x,y) ↔ F(u,v), then shifting the image by (x₀,y₀) in spatial domain:
f(x−x₀, y−y₀) ↔ F(u,v) · e^(−j2π(ux₀/M + vy₀/N))

Key implication: Translation only affects the phase of the Fourier transform, NOT its magnitude. So the power spectrum |F(u,v)|² is translation-invariant — moving an image doesn't change its frequency content appearance.

To center the DFT (move DC to center): multiply by (−1)^(x+y) before taking DFT:
f(x,y)·(−1)^(x+y) ↔ F(u−M/2, v−N/2)

Convolution Property of 2D DFT:
If f(x,y) ↔ F(u,v) and h(x,y) ↔ H(u,v), then:
f(x,y) ★ h(x,y) ↔ F(u,v) · H(u,v)

Conversely: f(x,y) · h(x,y) ↔ (1/MN) · F(u,v) ★ H(u,v)

Practical significance: Spatial domain convolution is computationally expensive O(N⁴) for an N×N image with N×N kernel. Using the convolution theorem, we can perform the same operation as: FFT(f) → multiply pointwise with FFT(h) → IFFT → result. This is O(N² log N), much faster. This is how all real-time image filtering works in practice.
Q. Prove whether the Fourier Transform of a discrete time signal is continuous or discrete. Explain with an example.
May 2025 Part B Q2 · 4 marks
Fourier Transform of a Discrete Signal:

The Fourier Transform (FT) of a continuous signal f(t) is continuous in frequency.
The Fourier Transform of a discrete sequence f[n] (DTFT) is: F(e^jω) = Σ f[n]·e^(-jωn) — this is continuous and periodic in ω (period 2π).
The DFT samples this DTFT at N equally spaced frequencies, giving a discrete frequency domain.

Example: Let f[n] = {1, 2, 3, 4} (N=4 point sequence)
DFT: F[0] = Σ f[n] = 1+2+3+4 = 10 (DC)
F[1] = 1·e⁰ + 2·e^(-jπ/2) + 3·e^(-jπ) + 4·e^(-j3π/2)
= 1 + 2(-j) + 3(-1) + 4(j) = (1-3) + j(-2+4) = -2 + 2j
|F[1]| = √(4+4) = 2√2

Conclusion: The DFT of a discrete N-point sequence gives N discrete complex frequency samples. The DTFT (theoretical) of a discrete signal is continuous and periodic; the DFT is a sampled, discrete version of the DTFT. So: Discrete signal → DFT → Discrete spectrum.

Frequency Domain Filters

85% · Part A/B

Frequency Domain Filtering Pipeline

Fig 5.1 — 5-step frequency domain filtering pipeline

⭐⭐⭐ Low-Pass Filters (Smoothing)

Low-pass filters pass low frequencies (smooth/DC) and attenuate high frequencies (edges/noise) → result is a smoothed/blurred image.

ILPF — Ideal Low-Pass
H(u,v) = 1 if D(u,v) ≤ D₀
H(u,v) = 0 if D(u,v) > D₀

Sharp cutoff. Causes ringing (Gibbs effect) in spatial domain. Not used in practice.

BLPF — Butterworth Low-Pass
H(u,v) = 1 / [1 + (D(u,v)/D₀)^(2n)]

n = order. Smooth transition. No ringing. n→∞ approaches ILPF.

GLPF — Gaussian Low-Pass
H(u,v) = e^(-D²(u,v)/2D₀²)

Gaussian shape. No ringing. Spatial counterpart is also Gaussian. Most natural blur.

Distance from Center
D(u,v) = √[(u - M/2)² + (v - N/2)²]
D₀ = cutoff frequency (radius in pixels). Larger D₀ → less smoothing. Smaller D₀ → more blur.

⭐⭐ High-Pass Filters (Sharpening)

High-pass filters attenuate low frequencies and pass high frequencies → enhances edges, sharpens image. Relationship: H_HP(u,v) = 1 - H_LP(u,v)

IHPF — Ideal High-Pass
H(u,v) = 0 if D(u,v) ≤ D₀
H(u,v) = 1 if D(u,v) > D₀

Sharp cutoff. Severe ringing. Not practical.

BHPF — Butterworth High-Pass
H(u,v) = 1 / [1 + (D₀/D(u,v))^(2n)]

Smooth transition. No ringing. Preferred.

GHPF — Gaussian High-Pass
H(u,v) = 1 - e^(-D²(u,v)/2D₀²)

No ringing. Smooth. Natural sharpening.

⚡
LP ↔ HP Relationship + Ringing Trick

HP = 1 - LP always! So IHPF = 1 - ILPF, BHPF = 1 - BLPF, GHPF = 1 - GLPF.
Ringing (Gibbs effect): Only IDEAL filters cause ringing. Butterworth and Gaussian do NOT. Why? Ideal filter has a sharp rectangular spectrum → its spatial domain response is a sinc function with infinite oscillations.
Exam answer: "Ideal filters cause ringing due to the sharp cutoff in frequency domain; Butterworth/Gaussian have smooth rolloff and therefore no ringing."

📝 RTU PYQs — Frequency Filters
Q. What is the difference between a box low pass filter and a Gaussian low pass filter?
June 2023 Part A Q3 · 2 marks
Box (Average) Low-Pass Filter vs Gaussian Low-Pass Filter:

Box Filter:
• Spatial: equal weights 1/9 in 3×3 neighborhood
• Frequency: Sinc-like spectrum with side lobes → causes ringing (Gibbs effect) at edges
• Not separable in a smooth way
• Fast to compute (just addition + division)

Gaussian Filter:
• Spatial: Gaussian-weighted kernel, center has highest weight
• Frequency: Also Gaussian (Fourier transform of Gaussian is Gaussian!) → NO ringing
• Separable: 2D Gaussian = 1D Gaussian × 1D Gaussian → even faster computation
• More natural blur (matches HVS point spread function)

Key Difference: The box filter's rectangular spatial kernel creates a sinc frequency response with significant side lobes, causing ringing artifacts. The Gaussian filter's smooth spatial kernel has a Gaussian frequency response with no side lobes, producing clean, artifact-free smoothing.
Q. Explain IHPF and Image Enhancement. How do frequency domain filters contribute to image transformation?
2022 Part B Q6 · May 2025 Part A Q7 · 2–4 marks
IHPF (Ideal High-Pass Filter):
H(u,v) = 0 if D(u,v) ≤ D₀; H(u,v) = 1 if D(u,v) > D₀
The IHPF completely blocks all frequencies within a circle of radius D₀ from the center (these are low frequencies = smooth areas). All frequencies outside D₀ (high frequencies = edges, fine detail) pass unchanged.

Effect: The result is an image containing primarily edges and texture — the smooth background becomes dark (black). Useful for edge detection in frequency domain.

Problem: Sharp cutoff → ringing artifacts (Gibbs effect) in the filtered image, especially near strong edges.

How Frequency Domain Filters Contribute to Image Transformation:
1. Low-pass filtering: Removes high-frequency noise → smooth/blur the image
2. High-pass filtering: Removes low-frequency smooth areas → enhances edges and sharpens
3. Band-pass/band-reject: Remove periodic noise patterns (remove specific frequency bands causing interference patterns)
4. Notch filtering: Remove specific periodic noise frequencies (e.g., 50Hz power line interference in MRI)

The convolution theorem (f★h ↔ F·H) makes frequency filtering computationally efficient — multiplying pointwise in frequency domain is much faster than spatial convolution.

Color Models

🔥 80% · Part B/C

Why Different Color Models?

Different applications need different ways to represent color. RGB is natural for displays; HSI is natural for human perception; CMY/CMYK is natural for printing. No single model serves all purposes.

⭐ 1. RGB Color Model

The RGB model represents color as a combination of Red, Green, and Blue primaries. It is an additive color model — adding all three at full intensity gives white.

RGB Cube
Each pixel: (R, G, B) ∈ [0, 255]³ for 24-bit color
Total colors: 256³ = 16.7 million. Black = (0,0,0). White = (255,255,255).
Primary colors: Red=(255,0,0), Green=(0,255,0), Blue=(0,0,255)
Secondary: Cyan=(0,255,255), Magenta=(255,0,255), Yellow=(255,255,0)

Problem with RGB for image processing: R, G, B are highly correlated (changing luminosity changes all three). Hard to process "color" independent of "brightness".

⭐⭐ 2. CMY/CMYK Model (Printing)

CMY is a subtractive model used in color printing. Inks SUBTRACT light from white paper.

RGB → CMY Conversion
[C] [1] [R/255]
[M] = [1] - [G/255]
[Y] [1] [B/255]
CMYK adds K (black) for efficiency: K = min(C,M,Y). C' = (C-K)/(1-K). Saves ink.

⭐⭐⭐ 3. HSI Color Model — Most RTU-Tested

The HSI model separates color information into three components that align with human perception:

  • H (Hue): The dominant color wavelength — what color it actually is (red, green, blue...). Range: [0°, 360°]
  • S (Saturation): How pure the color is — how far from gray. Range: [0, 1]. S=0 = gray.
  • I (Intensity): Overall brightness. Range: [0, 1]. Independent of H and S.
Fig 6.1 — HSI double-cone model: H=hue (angle), S=saturation (radius), I=intensity (height)

RGB → HSI Conversion (Must Memorize)

RGB → HSI (when R,G,B normalized to [0,1])
I = (R + G + B) / 3
S = 1 - [3/(R+G+B)] · min(R,G,B)
θ = arccos{ [(R-G)+(R-B)] / [2·√((R-G)²+(R-B)(G-B))] }
H = θ if B ≤ G
H = 360° - θ if B > G
📌 RGB → HSI Example

Given R=0.5, G=0.5, B=0 (a dark yellow/olive color)

I = (0.5+0.5+0)/3 = 0.333

S = 1 - [3/(0.5+0.5+0)] · min(0.5,0.5,0) = 1 - [3/1] × 0 = 1 - 0 = 1.0 (fully saturated)

θ: numerator = [(0.5-0.5)+(0.5-0)] / [2·√(...)] = 0.5/[2·√(0+0.25·0.5)]

= 0.5/[2·√0.125] = 0.5/0.707 = 0.707 → θ = arccos(0.707) = 45°

Since B=0 ≤ G=0.5: H = 45° (yellow-orange hue ✓)

HSI → RGB Conversion (Sector-Based)

HSI → RGB (for 0° ≤ H < 120° sector — RG sector)
B = I(1 - S)
R = I[1 + S·cos(H)/cos(60°-H)]
G = 3I - (R + B)
For other sectors: subtract 120° from H for GB sector, subtract 240° for BR sector. Same formulas with R,G,B roles rotated.

Why Process in HSI?

In RGB, you cannot modify brightness without affecting color. In HSI:

  • To enhance brightness: modify only the I channel
  • To change color: modify only H
  • To adjust color purity: modify only S
  • Histogram equalization → apply only to I channel → color-safe enhancement

Q. Explain HIS color model with an appropriate figure. Explain the conversion procedure from RGB color space to HSI color space.
2024 Part B Q3 · May 2025 Part B Q3 · June 2023 Part C Q4 · 4–10 marks ⭐ Repeated every year!
HSI Color Model:
The HSI (Hue-Saturation-Intensity) model describes color in terms of:
• H (Hue): Dominant wavelength / color type. Measured as angle 0°–360°. Red≈0°, Green≈120°, Blue≈240°.
• S (Saturation): Purity of color. 0=pure gray, 1=fully pure color.
• I (Intensity): Overall brightness, independent of color. I=0 → black, I=1 → white.

[Draw the HSI double cone: vertical I-axis from black (bottom) to white (top). At middle, a circle. Radius = saturation. Angle around circle = hue. Red at 0°, Green at 120°, Blue at 240°]

Advantage over RGB:
RGB components are highly correlated with luminance. In HSI, I is completely separated from H and S. This allows processing brightness and color independently.

RGB → HSI Conversion:
Given normalized R,G,B ∈ [0,1]:

Intensity: I = (R + G + B)/3

Saturation: S = 1 - [3/(R+G+B)] · min(R,G,B)
(S = 0 when R=G=B = gray)

Hue:
θ = arccos{ 0.5·[(R-G)+(R-B)] / √[(R-G)² + (R-B)(G-B)] }
H = θ if B ≤ G
H = 360° - θ if B > G

Example: R=1, G=0, B=0 (pure red)
I = 1/3 = 0.333
S = 1 - [3/1]×0 = 1 (fully saturated)
θ = arccos{0.5×[(1)+(1)]/√[1+0]} = arccos(1) = 0°
B=0 ≤ G=0, so H = 0° ✓ (red hue)

Applications of HSI:
1. Histogram equalization of I only → no color distortion
2. Color-based object detection using only H channel
3. Noise reduction in I channel without affecting hue
4. Skin detection using H and S ranges
Q. Explain following with example: (a) Lookup Table (b) Additive vs Subtractive color system (c) Hue and Saturation (d) Hue-Max-Min-Diff (HMMD) Color Space
May 2025 Part C Q4 · 10 marks
(a) Lookup Table (LUT):
A lookup table is a 1D array that maps each input gray level to an output gray level. Instead of computing the transform function for each pixel, simply look up the precomputed output.
Example: For gamma correction (γ=0.5), precompute LUT[r] = 255×(r/255)^0.5 for r=0..255. Then for each pixel, output = LUT[pixel_value]. Fast: O(1) per pixel.

(b) Additive vs Subtractive Color System:
• Additive (RGB): Colors made by adding light. Black = no light (0,0,0). White = all light (255,255,255). Used in displays, monitors, cameras. Primaries: Red, Green, Blue.
• Subtractive (CMY): Colors made by subtracting light from white paper. Black = maximum ink absorption. White = no ink. Used in printing. Primaries: Cyan, Magenta, Yellow.
Relationship: C=1-R, M=1-G, Y=1-B (complementary)

(c) Hue and Saturation:
Hue: The dominant color attribute. Described by angle in HSI model: 0°=Red, 120°=Green, 240°=Blue. Hue is what lets you distinguish "red" from "blue."
Saturation: Purity of the color. S=0 → achromatic (gray, no color). S=1 → pure, vivid color. Example: Pink = Red with low saturation (mixed with white).

(d) HMMD Color Space:
HMMD (Hue-Max-Min-Diff) is a color space derived from RGB:
H = Hue (same as HSI)
Max = max(R,G,B), Min = min(R,G,B)
Diff = Max - Min (similar to chroma/saturation measure)
Used in MPEG-7 color descriptors for compact color representation and efficient image retrieval. Better for image database search than RGB because it better separates luminance from chroma information.

Wavelet Transforms

70% · Part A

Motivation — Why Wavelets?

The Fourier Transform gives frequency information globally — it tells you which frequencies are present but not WHEN (WHERE in the image) they occur. For non-stationary signals (where frequency content varies with location), this is a limitation.

Wavelet Transform provides both time (spatial) and frequency information simultaneously — it can tell you which frequencies are present at which locations in the image.

🔗
Fourier vs Wavelet — Key Distinction

Fourier Transform: Basis functions are complex exponentials (sine/cosine) — infinite duration, global analysis.
Wavelet Transform: Basis functions are "wavelets" — localized, finite-duration oscillations. Can zoom in to specific locations AND frequencies simultaneously.
Analogy: FT is like listening to a whole song at once. WT is like reading a music score — you know what note plays at each moment.

Key Concepts

Mother Wavelet ψ(t)

The fundamental wavelet function. All wavelets used in analysis are scaled and shifted versions of the mother wavelet:

Scaled and Shifted Wavelets
ψₐ,ₜ(x) = (1/√a) · ψ((x-τ)/a)
a = scale parameter (1/a ≈ frequency). Small a → high frequency (zoom in). Large a → low frequency (zoom out).
τ = translation (position). 1/√a = normalization factor.

Continuous Wavelet Transform (CWT)

W(a,τ) = (1/√a) · ∫ f(x) · ψ*((x-τ)/a) dx
ψ* = complex conjugate of ψ. W(a,τ) = wavelet coefficient at scale a and position τ.

Discrete Wavelet Transform (DWT) — Used in Image Processing

The DWT decomposes an image into a set of sub-bands at different scales and orientations using filter banks:

Fig 7.1 — 2D DWT decomposes image into 4 sub-bands: LL (approximate) + LH+HL+HH (details)

Applications of Wavelet Transform in Image Processing

  • Image compression: JPEG 2000 uses DWT. Better quality at high compression vs JPEG (DCT).
  • Denoising: Threshold small wavelet coefficients (noise) while keeping large ones (signal).
  • Edge detection: High-frequency sub-bands (LH, HL, HH) correspond to edges.
  • Feature extraction: Multi-resolution analysis for texture recognition.
  • Medical imaging: Compact representation of MRI, CT scans.

Q. What is the significance of Wavelet Transforms?
May 2025 Part A Q8 · 2 marks
Significance of Wavelet Transforms:

1. Multi-resolution analysis: Wavelets represent images at multiple scales simultaneously — like seeing both the forest and the trees. This enables analysis of both coarse structure (low frequencies) and fine details (high frequencies) at their natural scales.

2. Joint time-frequency localization: Unlike Fourier transform (which gives global frequency information), wavelets provide BOTH location AND frequency information. A wavelet coefficient tells you "this frequency is strong at this position."

3. Image compression (JPEG 2000): DWT provides better compression than DCT (JPEG) because it analyzes the image at multiple scales. High-frequency detail coefficients can be aggressively quantized or discarded.

4. Image denoising: Noise appears as many small wavelet coefficients spread across all scales, while signal concentrates in few large coefficients. Thresholding small coefficients removes noise.

5. Edge detection: The LH, HL, HH sub-bands directly correspond to horizontal, vertical, and diagonal edges.
