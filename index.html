What This Course Aims to Teach

Digital Image Processing (6CS3-01 / 6E7101) is a core sixth-semester paper for CS, IT, AID, and CAI branches under RTU Kota. The course builds a complete understanding of how digital images are represented, enhanced, restored, compressed, and analysed using mathematical tools and algorithms.

By the end of this course, you will be able to:

  1. Understand how a physical scene becomes a digital image — sampling, quantization, and acquisition.
  2. Apply spatial and frequency-domain techniques to enhance image quality (Unit 3).
  3. Model degradation mathematically and restore blurred/noisy images (Unit 4).
  4. Implement lossless and lossy compression algorithms, understand Huffman, LZW, arithmetic coding, and JPEG (Unit 5).
  5. Segment images into meaningful regions using edge detectors, Hough transform, and region methods (Unit 6).
  6. Represent and describe image boundaries using chain codes and Fourier descriptors (Unit 6).

What is Digital Image Processing?

A digital image is a 2D function f(x, y) where (x, y) are spatial coordinates and f is the intensity (brightness) at that point. When both coordinates and intensity are finite and discrete — it is a digital image stored as a matrix (2D array) of pixels.

Digital Image Processing (DIP) is the use of computer algorithms to manipulate digital images. Unlike analogue image processing, digital methods are precise, reproducible, and can be highly automated.

Digital Image — Mathematical Definition
f(x, y) where x ∈ {0, 1, …, M−1}, y ∈ {0, 1, …, N−1}
f(x,y) ∈ {0, 1, …, L−1} where L = 2ᵏ (k = bits per pixel)
M × N = image dimensions. L = number of gray levels. 8-bit image: L = 256.

Three levels of processing:

  • Low-level: Input = Image → Output = Image. Enhancement, restoration, compression. (Units 3, 4, 5)
  • Mid-level: Input = Image → Output = Features. Segmentation, boundary detection. (Unit 6)
  • High-level: Input = Features → Output = Meaning. Object recognition, AI classification. (Beyond syllabus)

10-Mark Answer Framework (Part C)

  1. Introduction — Define the topic clearly (2–3 lines).
  2. Mathematical Formulation — Write the key formula in a box (3–4 marks from this).
  3. Algorithm/Steps — Numbered list of procedure.
  4. Diagram — Labelled block diagram or kernel (3–4 marks standalone!).
  5. Example/Numerical — Demonstrates understanding.
  6. Advantages, Limitations, Applications — Fills remaining space.
⚡
Diagram Rule — Guaranteed 3 Marks

RTU examiners award marks specifically for diagrams. A labelled block diagram always earns 3–4 marks even if your written answer is incomplete. For every Part C question — draw the system block diagram first. For edge detection: draw the mask/kernel. For JPEG: draw the 6-stage pipeline. For homomorphic filtering: draw the ln→DFT→H(u,v)→IDFT→exp pipeline. Never skip the diagram.
