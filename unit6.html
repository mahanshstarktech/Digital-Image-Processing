01

What is Image Segmentation?

Conceptual

Definition

Image Segmentation is the process of partitioning a digital image into multiple regions (sets of pixels) such that pixels in the same region share similar properties (intensity, color, texture), and pixels in different regions are dissimilar.

In simple words: You have a photo of a cat sitting on a sofa. Segmentation separates the image into — cat pixels, sofa pixels, and background pixels.

Fig 1.1 — Segmentation partitions image into meaningful regions

Why Segmentation? (Write this for 2-mark "need of segmentation" questions)

  • Object recognition: Identify and locate objects in an image (e.g., face detection).
  • Medical imaging: Segment tumors, organs, or tissues from MRI/CT scans.
  • Machine vision: Industrial inspection systems to detect defects.
  • Satellite imaging: Separate land, water, vegetation regions.
  • Traffic systems: Detect lanes, vehicles, pedestrians.

Approaches to Segmentation (The Big 3 — Write this as intro in any Part-C answer)

  • Discontinuity-based: Find abrupt changes in intensity → Edge detection, Point/Line detection
  • Similarity-based: Group pixels with similar properties → Thresholding, Region Growing
  • Boundary-based: Detect and link boundaries → Hough Transform, Boundary linking

02

Point & Line Detection

70% · Part A/B

Basic Concept: What are Discontinuities?

In an image, discontinuities are locations where the intensity changes abruptly. There are three types:

  • Point: An isolated pixel with very different intensity from neighbors (isolated bright/dark dot)
  • Line: A thin, elongated set of pixels with different intensity (scratch on metal, wire, etc.)
  • Edge: A broader boundary between two regions (most important!)

Detection is done using masks (kernels) — you slide the mask over the image and check where the response exceeds a threshold.

Point Detection

A point is detected using the Laplacian mask. If the response |R| at a pixel exceeds threshold T, a point is detected there.

Response of mask w at point (x,y)
R = w₁z₁ + w₂z₂ + w₃z₃ + ... + w₉z₉ = Σ wᵢzᵢ
where z₁ to z₉ = pixel values in the 3×3 neighborhood, w = mask coefficients
A point is detected if: |R| ≥ T

Q. Explain gradient operator?
June 2022 · Part A Q5 · 2 marks
The gradient operator measures the rate of change of intensity in an image. For a 2D image f(x,y):

∇f = [∂f/∂x, ∂f/∂y] = [Gx, Gy]

Magnitude: |∇f| = √(Gx² + Gy²) ≈ |Gx| + |Gy| (approximation used in practice)
Direction: θ = arctan(Gy/Gx)

The gradient points in the direction of maximum intensity increase. Gradient magnitude is high at edges and low in uniform regions. Used in edge detection algorithms like Sobel, Prewitt, and Roberts.
03

Edge Detection

🔥 95% — Most Important

What is an Edge?

An edge is a set of connected pixels that lie on the boundary between two regions. Edges occur where there is a sharp change in intensity. They contain the most important structural information about an image — shapes, boundaries, and texture.

Fig 3.1 — Edge = sharp intensity change → 1st derivative peaks at edge location

The Gradient — Mathematical Foundation

For a 2D image f(x,y), the gradient vector is:

Gradient Vector
∇f = [∂f/∂x, ∂f/∂y]ᵀ = [Gx, Gy]ᵀ
Magnitude: M(x,y) = |∇f| = √(Gx² + Gy²) ≈ |Gx| + |Gy|
Direction: α(x,y) = arctan(Gy / Gx)
The gradient direction is perpendicular to the edge direction.
⚡
Fast Calculation Trick

In numerical problems, use the approximation M ≈ |Gx| + |Gy| instead of √(Gx²+Gy²). This avoids square root and gives nearly the same result. RTU expects this. Example: Gx=3, Gy=4 → exact=5, approx=7. For direction: θ = arctan(4/3) ≈ 53°.

Important Limitation of Laplacian

Laplacian is very sensitive to noise (amplifies noise because it takes second derivative). Solution: Apply Gaussian smoothing FIRST, then apply Laplacian → this gives the LoG (Laplacian of Gaussian) operator, also called "Mexican Hat" filter.

⭐⭐ LoG — Laplacian of Gaussian (Marr-Hildreth)

Combines Gaussian smoothing with Laplacian edge detection. Best of both worlds: smooth noise, then find edges.

LoG Formula (Mexican Hat)
LoG(x,y) = -1/(πσ⁴) · [1 - (x²+y²)/(2σ²)] · e^(-(x²+y²)/(2σ²))
σ = standard deviation of Gaussian; Controls the scale of edge detection
Zero-crossings of LoG output = edge locations

Q. What are the derivative operators useful in image segmentation? Explain their role in segmentation.
2024 Part B Q5 & Part C Q4(b) · 4–10 marks
Derivative Operators in Image Segmentation

Segmentation based on discontinuity uses derivative operators to detect abrupt changes in intensity (edges, points, lines).

1. First-Order Derivative (Gradient) Operators:
These compute the gradient ∇f = [Gx, Gy]. Edges appear as local maxima in the gradient magnitude M = √(Gx²+Gy²).

a) Roberts Operator (2×2):
Gx = z₅−z₉, Gy = z₆−z₈. Simple but very sensitive to noise.

b) Prewitt Operator (3×3):
Uses equal-weight averaging in smoothing direction. Gx detects vertical edges, Gy detects horizontal edges.

c) Sobel Operator (3×3):
Improved Prewitt with center weight doubled (2 instead of 1). Better noise suppression. Most widely used. [Draw all three kernels — Gx and Gy for each]

2. Second-Order Derivative (Laplacian) Operators:
∇²f = ∂²f/∂x² + ∂²f/∂y²
Edges are located at zero-crossings of the second derivative. Laplacian mask (4-neighbor): center=-4, cross=+1.
LoG combines Gaussian smoothing with Laplacian to handle noise.

Role in Segmentation:
• These operators highlight boundaries between image regions.
• The detected edges form the boundary of segments.
• After edge detection, edge linking and thresholding are applied to complete segment boundaries.
• Combined with thresholding: pixels with M ≥ T are classified as edge pixels; these form segment boundaries.

04

Thresholding

80% · Part B/C

What is Thresholding?

Thresholding is the simplest and most widely used image segmentation method. It converts a grayscale image into a binary image by comparing each pixel's intensity to a threshold value T.

Basic Thresholding
g(x,y) = 1 if f(x,y) > T
g(x,y) = 0 if f(x,y) ≤ T
Result is a binary image: 1 = object (foreground), 0 = background
Fig 4.1 — Bimodal histogram: threshold T placed in valley separates background from object

Types of Thresholding

1. Global Thresholding (Single T for entire image)

One threshold applied to the whole image. Works well when the histogram is bimodal (two clear peaks with a valley).

Example: Document scanning — text is dark (object), paper is white (background). Clear bimodal histogram.

2. Optimal (Otsu's) Thresholding

Automatically finds the best threshold T by maximizing the between-class variance (variance between foreground and background classes).

Otsu's Method
σ²_B(T) = ω₁(T)·ω₂(T)·[μ₁(T) - μ₂(T)]²
ω₁, ω₂ = class probabilities; μ₁, μ₂ = class means
Find T* = arg max [σ²_B(T)]

3. Local (Adaptive) Thresholding

Different thresholds used for different regions of the image. Useful when illumination is non-uniform. T(x,y) varies across the image.

Example: A document photographed under uneven light — different T needed for top and bottom portions.

4. Multiple Thresholding

Two or more thresholds used to segment image into multiple classes.

g(x,y) = a if f(x,y) > T₂
g(x,y) = b if T₁ < f(x,y) ≤ T₂
g(x,y) = c if f(x,y) ≤ T₁
🔗
Connection: Thresholding ↔ Unit 3 Histogram Processing

Thresholding is directly related to histogram processing (Unit 3). The histogram shape tells you where to place T. A bimodal histogram (two humps) = easy thresholding. A flat/unimodal histogram = use Otsu's. Histogram equalization (Unit 3) can be applied BEFORE thresholding to improve results on poorly-contrasted images.

📝 RTU PYQs — Thresholding
Q. Discuss image segmentation based on various thresholding techniques.
May 2025 · Part B Q7 · 4 marks
Image Segmentation by Thresholding

Thresholding converts a grayscale image f(x,y) to binary g(x,y) based on a threshold T.

1. Global Thresholding:
Single T applied to entire image. g(x,y) = 1 if f(x,y) > T, else 0.
Steps: (a) Choose initial T = mean intensity. (b) Segment image using T. (c) Compute means μ₁, μ₂ of both groups. (d) Update T = (μ₁+μ₂)/2. (e) Repeat until T converges.

2. Otsu's Optimal Thresholding:
Selects T that maximizes between-class variance σ²_B = ω₁ω₂(μ₁-μ₂)². Automatic, no manual selection needed.

3. Local/Adaptive Thresholding:
T varies with location. T(x,y) computed for each pixel using local neighborhood statistics (local mean or Gaussian-weighted mean). Handles uneven illumination.

4. Multiple Thresholding:
Two thresholds T₁, T₂ create three classes. Used in images with multiple distinct objects.

Application: Document image binarization, medical image segmentation (separating lesions from tissue).

Edge & Boundary Linking

75% · Part B

Why Edge Linking?

Edge detection operators produce pixels that ideally correspond to edges. But in practice, due to noise, the output is broken, disconnected edge segments. Edge linking joins these fragments to form continuous, closed boundaries.

Local Processing (Local Edge Linking)

Look at a small neighborhood around each edge pixel (x,y). Link it to a neighboring edge pixel (s,t) if both conditions hold:

Local Linking Criteria
Condition 1 (Magnitude): |M(x,y) - M(s,t)| ≤ E (similar strength)
Condition 2 (Direction): |α(x,y) - α(s,t)| ≤ A (similar orientation)
E = magnitude tolerance, A = angle tolerance (typically 45°). Both must be satisfied to link pixels.
📌 Example

Pixel P1 at (x,y): M=45, α=30°. Pixel P2 at (x+1,y): M=43, α=35°.
With E=5, A=15°: |45-43|=2 ≤ 5 ✓, |30-35|=5° ≤ 15° ✓ → Link P1 and P2.

Global Processing — Graph-Based Edge Linking

Represents edge pixels as nodes in a graph and finds the best path (boundary) through cost-function minimization. More powerful than local linking but computationally expensive.

Steps:

1
Convert image to a directed graph. Each pixel = a node. Edge cost = (1 - gradient magnitude) — high gradient = low cost.
2
Apply dynamic programming or Dijkstra's algorithm to find the minimum cost path from start to end node.
3
The minimum cost path = the optimal boundary of the object.
📝 RTU PYQs — Edge Linking
Q. How do you link edge pixels through global processing? Also explain Hough Transforms.
2024 · Part B Q7 · 4 marks
Global Edge Linking via Graph Methods:

After edge detection, the output contains many isolated edge segments. Global processing uses a graph-theoretic approach:
1. Create a directed weighted graph where each edge pixel is a node.
2. Assign cost c(i,j) to each edge between nodes. Low cost = high gradient = likely edge. c(i,j) = G_max - M(i,j).
3. Apply dynamic programming to find the minimum cost path from a start point to an end point.
4. The resulting path represents the linked boundary.

Advantage: Considers global image information, not just local neighborhoods. Finds globally optimal boundaries.

Hough Transform: (See next section for full details)
Hough Transform is a global technique that detects lines, circles, or other parametric shapes by transforming the problem to a parameter (accumulator) space. Each edge pixel votes for all possible lines passing through it. Lines with the most votes = actual lines in the image.

Hough Transform

🔥 90% — Very Important

Core Idea — Parameter Space Voting

The Hough Transform detects shapes (lines, circles, ellipses) in an image by using a voting procedure. Instead of working in image (x,y) space, it works in parameter space.

Key insight: All lines passing through a point (x₀, y₀) satisfy y₀ = mx₀ + c for different (m,c) pairs. In (m,c) parameter space, this point (x₀,y₀) maps to a line!

Hough Transform for Lines — Normal Form (Preferred)

The slope-intercept form y = mx + c has problems with vertical lines (m → ∞). Instead, use normal (Hessian) form:

Normal Form of a Line (Used in Hough)
ρ = x·cosθ + y·sinθ
ρ = perpendicular distance from origin to line
θ = angle of the perpendicular from x-axis (0° to 180°)
Range: -D ≤ ρ ≤ D where D = diagonal length of image
Fig 6.1 — Three collinear points map to sinusoids in (θ,ρ) space; their intersection gives the line parameters

Algorithm — Hough Transform Steps

1
Edge Detection: Apply Sobel/Canny to get binary edge image.
2
Create Accumulator Array: H[ρ, θ] = 0 (2D array over quantized ρ and θ values). This is also called the Hough space.
3
Vote: For each edge pixel (x,y), for each θ from 0° to 180°, compute ρ = x·cosθ + y·sinθ, then increment H[ρ,θ]++.
4
Find Peaks: Find local maxima in H[ρ,θ]. Each peak (ρ*,θ*) = a detected line in the image.
5
Reconstruct: Convert (ρ*,θ*) back to image space using ρ = x·cosθ + y·sinθ.

Limitations of Hough Transform

  • Computationally expensive for complex shapes (large accumulator arrays)
  • Quantization error in accumulator can cause missed or spurious detections
  • For circles/ellipses, parameter space becomes 3D or 5D — very costly
  • Cannot detect shapes with no known parametric model
  • Sensitive to noise in the edge detection step
🧠
Exam Trick — Collinearity Check with Hough

To check if points are collinear using Hough: For each point, compute ρ = x·cosθ + y·sinθ for several θ values. Plot in (θ,ρ) space. If all curves intersect at ONE point → collinear. If no common intersection → not collinear. This is a standard RTU numerical!

📝 RTU PYQs — Hough Transform
Q. Discuss Hough transformation concept + check if points (1,0), (2,1), (3,1), (4,1), (3,2) are collinear. Also mention limitations.
June 2023 · Part C Q3 · 10 marks ⭐ Most Important
Hough Transform — Concept:
The Hough Transform detects lines (and shapes) by mapping image space to parameter space. For a line ρ = x·cosθ + y·sinθ, each image point (x,y) contributes a sinusoidal curve in (θ,ρ) space. Collinear points produce curves that intersect at the same (θ*,ρ*), which is the Hough parameter of the line containing them.

Collinearity Check for Given Points:
Use y = mx + c form: If all points lie on y = mx + c, then in (m,c) space, each point gives a line c = -mx + y, and all lines intersect at (m*,c*).

For points: (1,0), (2,1), (3,1), (4,1), (3,2)
Check (1,0): c = -m + 0 → c = -m
Check (2,1): c = -2m + 1
Intersection of these two: -m = -2m + 1 → m = 1, c = -1
Line: y = x - 1

Verify all points on y = x - 1:
(1,0): 0 = 1-1 = 0 ✓
(2,1): 1 = 2-1 = 1 ✓
(3,1): 1 = 3-1 = 2 ✗ (Not on line!)
(4,1): 1 = 4-1 = 3 ✗
(3,2): 2 = 3-1 = 2 ✓

Conclusion: Points (1,0), (2,1), (3,2) are collinear on y = x-1. Points (3,1) and (4,1) are NOT on this line → All 5 points are NOT collinear.

Limitations of Hough Transform:
1. Computationally expensive — large accumulator arrays needed
2. Quantization errors in discretizing (ρ,θ) space reduce accuracy
3. For circles: 3D parameter space (a,b,r) — very memory-intensive
4. Noise in edge detection step severely affects performance
5. Cannot detect non-parametric shapes
6. Difficulty with closely spaced parallel lines
Q. What is the role of Hough Transform in global processing?
May 2025 · Part A Q4 · 2 marks
Hough Transform is a global edge-linking technique that detects lines/shapes by transforming image space to parameter (ρ,θ) space. Each edge pixel votes for all possible lines through it. The accumulator array H[ρ,θ] counts votes. Local maxima in H correspond to actual lines — linking all contributing edge pixels globally into continuous boundaries. It connects isolated edge points that belong to the same geometric structure, providing a complete global boundary description.

Region Based Segmentation

🔥 85% · Part B/C

Concept

Region-based methods group pixels with similar properties into regions. Unlike edge detection (which finds boundaries), region methods directly find homogeneous areas.

Predicate P(R): A logical condition that must be true for all pixels in a region. Example: P(R) = "all pixels in R have intensity difference ≤ 10".

Segmentation Conditions
① ∪Rᵢ = Image (regions cover entire image)
② Rᵢ ∩ Rⱼ = ∅ for i ≠ j (regions don't overlap)
③ P(Rᵢ) = TRUE for all i (each region satisfies predicate)
④ P(Rᵢ ∪ Rⱼ) = FALSE for adjacent Rᵢ, Rⱼ (merging would violate predicate)
⑤ Each region Rᵢ must be connected

⭐⭐ Region Growing

Starts from one or more seed pixels and grows the region by adding neighboring pixels that satisfy the predicate.

Fig 7.1 — Region Growing: starts from seed, expands to similar neighbors

Seed Selection:

  • Manual selection by user
  • All pixels above/below a certain intensity
  • Small homogeneous areas found by initial scan

Stopping Criteria:

  • No more neighboring pixels satisfy the predicate
  • Region reaches a certain size
  • Predicate based on intensity difference, texture, color similarity

⭐⭐ Region Splitting and Merging

Combines a top-down split approach with a bottom-up merge approach. Often implemented using a Quadtree data structure.

Fig 7.2 — Split & Merge: regions failing predicate are split; adjacent uniform regions are merged

Algorithm:

S
Split Phase: If P(R) = FALSE for any region R, split it into 4 equal quadrants. Repeat recursively until P = TRUE for all regions or regions become single pixels.
M
Merge Phase: For any two adjacent regions Rᵢ, Rⱼ, if P(Rᵢ ∪ Rⱼ) = TRUE, merge them. Repeat until no more merges are possible.
💡
Key Difference — Growing vs. Splitting

Region Growing = bottom-up (starts from pixels, builds up). Splitting = top-down (starts from whole image, breaks down). Split+Merge = combines both for best results. RTU often asks you to "compare" — use this distinction as your answer core.

📝 RTU PYQs — Region Segmentation
Q. Explain the fundamental steps of region growing and region splitting algorithms in region-based segmentation.
June 2023 · Part B Q6 · 4 marks
Region Growing Algorithm:
1. Seed Selection: Choose initial seed pixels (manually or automatically).
2. Initialization: Mark seed pixels as the starting region R.
3. Growing: Examine all pixels neighboring R. If a neighbor pixel satisfies predicate P (e.g., |f(neighbor) - f(seed)| ≤ threshold T), add it to R.
4. Repeat: Continue until no more pixels can be added.
5. Label: All pixels in R belong to one segment. Unvisited pixels may be new seeds for other regions.

Key consideration: Seed selection is critical. Predicate P determines what constitutes "similar" pixels.

Region Splitting Algorithm (using Quadtree):
1. Start with the entire image as one region R.
2. If P(R) = FALSE (region is not homogeneous), split R into 4 equal quadrants Q₁, Q₂, Q₃, Q₄.
3. Apply step 2 recursively to each quadrant.
4. Stop splitting when P(Qᵢ) = TRUE or region = single pixel.
5. Merge step: For adjacent regions Rᵢ, Rⱼ: if P(Rᵢ ∪ Rⱼ) = TRUE, merge them.
6. Repeat merge until no more adjacent regions can be merged.

Result: Image partitioned into maximal homogeneous regions. Over-segmentation may occur, which the merge step corrects.

Boundary Representation

75% · Part B

Why Represent Boundaries?

After segmentation, we need to represent and describe the boundaries of regions for further analysis (shape recognition, pattern matching, compression). Two main approaches: Chain codes and Polygonal approximation.

⭐⭐ Chain Code (Freeman Chain Code)

Represents a boundary by a connected sequence of directional codes. Each step from one boundary pixel to the next is encoded as a number (0–7) indicating the direction.

Fig 8.1 — Chain code example: hexagonal boundary encoded as 1,0,7,5,4,3
4-connectivity codes → directions
0 = right (+x), 1 = up (+y), 2 = left (-x), 3 = down (-y)
8-connectivity codes → directions
0=E, 1=NE, 2=N, 3=NW, 4=W, 5=SW, 6=S, 7=SE
⚡
Rotation Invariance Trick

To make chain code rotation invariant: Compute the first difference of the chain code (subtract consecutive pairs mod 8). This gives the same result regardless of rotation. Example: Code 1,0,7,5,4,3 → differences (0-1)mod8=7, (7-0)mod8=7, (5-7)mod8=6, (4-5)mod8=7, (3-4)mod8=7, (1-3)mod8=6 → first diff = 7,7,6,7,7,6

⭐ Polygonal Approximation

Represents a boundary by a polygon — a set of straight-line segments. The goal is to use the minimum number of line segments while keeping approximation error below a threshold.

Key idea: A smooth curve/boundary is approximated by a polygon. The vertices of the polygon are selected boundary points.

Minimum Perimeter Polygon (MPP):

Enclose boundary in a grid of small cells (concentric squares). The MPP is the polygon of minimum perimeter that passes through the grid, fitting inside outer cell edges.

Merging Techniques:

  • Start with all boundary pixels as vertices.
  • Compute the line between consecutive vertices.
  • If all intermediate points are within ε distance from the line → merge (remove intermediate vertices).
  • Repeat until no more merges possible.

Boundary Descriptors

60% · Short Answers

What are Boundary Descriptors?

After representing a boundary, we compute descriptors — numerical features that characterize the shape. These are used for object recognition, classification, and matching.

Simple Descriptors

1. Boundary Length

Number of pixels in the boundary (for digital boundaries). For chain codes: count the codes. Diagonal steps contribute √2 times more than horizontal/vertical steps.

Length = n_h + n_v + √2 · n_d
n_h/n_v = horizontal/vertical steps (4-connected codes), n_d = diagonal steps

2. Diameter

The maximum distance between any two boundary points.

Diam(B) = max[D(pᵢ, pⱼ)] for all pᵢ, pⱼ ∈ B
The two points achieving maximum = major axis of the boundary

3. Eccentricity (Elongation)

Eccentricity = Major axis / Minor axis
Circle: eccentricity = 1. Elongated shape: eccentricity >> 1.

4. Curvature

Rate of change of slope along the boundary. High curvature = sharp corners. Low curvature = smooth curves.

Fourier Descriptors ⭐

Represent boundary shape using Fourier transform coefficients. Powerful because it captures shape at different scales.

Fourier Descriptor Computation
Represent boundary as complex sequence: s(k) = x(k) + j·y(k), k=0,1,...,K-1
DFT: a(u) = (1/K) · Σ s(k) · e^(-j2πuk/K)
First few a(u) = low-frequency components = coarse shape
Higher a(u) = high-frequency components = fine detail
Using only first D coefficients reconstructs a smoothed version of the boundary. Scale/rotation invariant variants exist.
🔗
Connection: Fourier Descriptors ↔ Unit 3 (Fourier Transform)

Fourier descriptors directly apply the DFT concepts from Unit 3. The boundary is treated as a 1D signal (parameterized by arc length or pixel index), and DFT gives frequency components that describe the shape. Low frequencies → overall shape. High frequencies → sharp corners and fine details. This is exactly the frequency domain analysis you learned in Unit 3!
