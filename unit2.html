What is Digital Image Processing?

65% · Conceptual Intro

Definition

Digital Image Processing (DIP) is the use of computer algorithms to process digital images — to improve their quality, extract useful information, or transform them for specific applications.

Three levels of processing:

  • Low-level processing: Input = image, Output = image. Noise removal, sharpening, contrast enhancement. (Unit 3 & 4)
  • Mid-level processing: Input = image, Output = features (edges, regions). Segmentation, object recognition. (Unit 6)
  • High-level processing: Input = features, Output = meaning. Scene understanding, AI, classification.

Applications of DIP (Write at least 5 for 2-mark answer)

  • Medical Imaging: CT, MRI, X-ray enhancement; tumor detection; segmentation of organs
  • Remote Sensing: Satellite imagery for land use mapping, disaster monitoring, weather forecasting
  • Industrial Inspection: Automated defect detection in manufacturing, PCB inspection
  • Forensics: Fingerprint analysis, face recognition, document forgery detection
  • Astronomy: Enhancing images from telescopes, removing atmospheric distortion
  • Military: Missile guidance, surveillance, target recognition
  • Entertainment: Film special effects, image/video compression (JPEG, MPEG)
  • Document Processing: OCR (Optical Character Recognition), barcode reading

Digital Image Representation

85% · Part A/B

What is a Digital Image?

A digital image is a 2D function f(x, y) where x and y are spatial (plane) coordinates and f is the amplitude (intensity or gray level) at that point. When x, y, and f are all finite discrete quantities, the image is a digital image.

Digital Image as a 2D Function
f(x, y) : pixel value at row x, column y
Image size: M rows × N columns = M×N pixels
Pixel value range: [0, L-1] where L = 2ᵏ (k = number of bits)
For 8-bit image: L = 256, range [0, 255]. 0 = black, 255 = white.
Fig 2.1 — A digital image is a 2D matrix of pixel values. f(x,y) = intensity at row x, column y.

Key Parameters of a Digital Image

1. Spatial Resolution (Image Size)

The number of pixels per unit area. Common resolutions: 640×480 (VGA), 1920×1080 (Full HD), 4096×2160 (4K).

Resolution & Storage
Total pixels = M × N
Storage (grayscale) = M × N × k bits
Storage (color RGB) = M × N × 3k bits
Example: 1024×1024 8-bit grayscale = 1024 × 1024 × 8 = 8,388,608 bits = 1 MB

2. Gray-Level Resolution (Bit Depth)

The number of bits used to represent the intensity at each pixel. Determines how many distinct gray levels exist.

Bit Depth (k)Gray Levels (L = 2ᵏ)Application
1 bit2 (binary)Text, fax, binary segmentation
2 bits4Very coarse (used in old displays)
4 bits16Thumbnail, icon-level
8 bits256Standard grayscale (most common)
12 bits4096Medical imaging (CT, MRI)
16 bits65536Professional photography, astronomy
24 bits (3×8)16.7M colorsTrue color RGB image
🧠
Memory Formula — "M × N × k"

Always remember: Storage = M × N × k bits. Converting to bytes: divide by 8. To KB: divide by 8192. To MB: divide by 8,388,608.
For 8-bit 1024×1024: = 1024×1024×8 = 8,388,608 bits = 1,048,576 bytes = 1 MB. This number appears often in RTU questions about storage requirements!

Pixel Relationships (Important for Segmentation in Unit 6)

Connectivity

Two pixels p and q are:

  • 4-connected: If they are horizontal or vertical neighbors (up, down, left, right). The 4 direct neighbors form the N₄(p) neighborhood.
  • 8-connected: If they include diagonal neighbors too. The 8 neighbors form the N₈(p) neighborhood.
  • D-neighbors (diagonal): Only the 4 diagonal pixels: NE, NW, SE, SW.
Fig 2.2 — 4-connectivity (direct neighbors) vs 8-connectivity (includes diagonals)

Digital Representation of Color Images:

A color image is represented as a stack of three 2D matrices (channels), one for each primary color component. The most common model is RGB (Red, Green, Blue).

For a 24-bit true-color image of size M×N:
• Each channel = M×N matrix of 8-bit values (0–255)
• Total: 3 × M × N × 8 bits
• Example: 512×512 color image = 3 × 512 × 512 × 8 = 6,291,456 bits = 768 KB

Color pixel at (x,y) = (R(x,y), G(x,y), B(x,y))
Total colors possible: 256³ = 16,777,216 ≈ 16.7 million colors

Other representations:
• HSI: Hue-Saturation-Intensity — separates color from brightness
• CMY: Cyan-Magenta-Yellow — for printing (subtractive model)
• YCbCr: Luminance + chrominance — used in JPEG and video compression

Sampling & Quantization

🔥 90% · Part A/B with Numerical

From Continuous to Digital — The Two-Step Process

A real-world scene produces a continuous image — it has infinite spatial detail (every position has a value) and infinite intensity precision. To store it in a computer, we must convert it to a digital image through two steps:

  1. Sampling — Discretizing the spatial coordinates (x, y). Determines M×N (resolution).
  2. Quantization — Discretizing the intensity values f. Determines number of gray levels L = 2ᵏ.
Fig 3.1 — Continuous scene → Sampling (spatial grid) → Quantization (intensity levels) → Digital image

⭐⭐⭐ Sampling — Spatial Discretization

Sampling takes a continuous image f(x,y) and measures its value at a discrete set of points on a regular grid. The denser the grid (more samples), the more detail is captured.

Nyquist-Shannon Sampling Theorem
f_s ≥ 2 · f_max
f_s = sampling frequency, f_max = highest spatial frequency in image.
If f_s < 2·f_max → ALIASING occurs (high frequency content appears as low frequency — artifacts!)
To avoid aliasing: either increase f_s (sample more densely) or pre-filter (remove high freq before sampling).

Effect of Reducing Sampling Rate

If you take an M×N image and reduce to M/2 × N/2, you lose spatial detail. Objects appear blocky — this is called pixelization.

📌 Sampling Calculation Example

An image captures a scene at 512×512 pixels. Required storage = 512×512×8 = 2,097,152 bits = 256 KB.

If we reduce to 256×256 (half the sampling rate): 256×256×8 = 524,288 bits = 64 KB. (4× size reduction)

But spatial resolution is halved → fine details like thin lines disappear → pixelization visible at edges.

⭐⭐⭐ Quantization — Intensity Discretization

Quantization maps the continuous intensity range [0, f_max] to a finite set of L = 2ᵏ discrete levels. Each pixel gets assigned to the nearest level.

Uniform Quantization Step Size
Δ = f_max / (L - 1) = f_max / (2ᵏ - 1)
Each level covers a range of Δ. Intensity values within Δ/2 of a level all map to that level.
Quantization error (noise) = f(x,y) − quantized value. Maximum error = ±Δ/2.

Effect of Reducing Bit Depth

Reducing the number of bits from k to fewer bits reduces the number of gray levels. With very few levels, smooth gradients appear as sudden jumps. This artifact is called false contouring (or quantization contouring).

Fig 3.2 — Effect of bit depth on image quality. Less bits → false contouring artifact.

⭐ Storage vs Quality — The Trade-off Table

ChangeEffect on ImageEffect on StorageArtifact if too extreme
↑ M, N (more samples)More spatial detail, sharper↑ StorageNo artifact (more is better)
↓ M, N (fewer samples)Loss of fine details, blocky↓ StoragePixelization, Aliasing
↑ k (more bits)Smoother tonal gradients↑ StorageNo artifact
↓ k (fewer bits)Fewer gray levels, steps visible↓ StorageFalse Contouring

2-Mark Question Shortcut

"What happens when you reduce sampling rate?" → Pixelization (blocky appearance, loss of spatial detail, aliasing)
"What happens when you reduce bit depth?" → False contouring (smooth gradients show as sudden jumps, fake edge appearance)

RTU PYQ Numerical — Sampling & Quantization Effects

📌 2024 Part B Q1 — Sampling and Quantization Operations on Image

Q: Explain what you would expect the result of Sampling and Quantization operations on image.


Sampling Operation — Effects:

Consider a continuous image of a circle on a grid. At high sampling rate (512×512), the circle appears smooth. At lower rates:

512×512 (high): Circle edge smooth, fine details visible
256×256 (medium): Some smoothing at edges
128×128 (low): Staircase effect visible at curved edges
 64×64 (very low): Object shape barely recognizable
 32×32 (extreme): Heavy pixelization, object looks blocky

→ Pixelization artifact: Individual pixels become visible as blocks. Curved edges appear jagged (staircase effect). Fine details (thin lines, text) disappear.


Quantization Operation — Effects:

256 gray levels (8-bit): Smooth tonal transitions in gradients
 64 gray levels (6-bit): Near-smooth, slight banding
 16 gray levels (4-bit): Visible banding in smooth regions
  8 gray levels (3-bit): Prominent bands (false contours)
  4 gray levels (2-bit): Strong false contouring
  2 gray levels (1-bit): Binary image, all detail lost

→ False contouring artifact: In areas with smooth gradients (skin, sky, smooth surfaces), abrupt intensity jumps create artificial-looking contour lines. More prominent in smooth, low-texture regions.


Key Rule: For typical images, reducing M,N affects spatial detail before intensity artifacts appear. Reducing k below ~4 bits causes noticeable false contouring. The human eye requires approximately 50–100 gray levels to perceive smooth gradients — below 6 bits (64 levels) is where false contouring becomes perceptible.

📝 RTU PYQs — Sampling & Quantization
Q. Define pixelization and false contouring effect in images.
June 2023 · Part A Q1 · 2 marks ⭐ Direct definition question
Pixelization:
Pixelization is the visual artifact that occurs when the sampling rate of an image is too low (too few pixels). When a low-resolution digital image is displayed at large size, individual square pixels become visible, making the image appear blocky or "mosaic-like." Curved edges and fine details appear as staircase-shaped (jagged) patterns.
Cause: Insufficient spatial sampling. Each pixel covers a large area of the original scene.
Example: Zooming in on a low-resolution photo — the image becomes a grid of colored squares.

False Contouring Effect:
False contouring (quantization contouring) is the visual artifact that occurs when the number of quantization levels (bit depth) is too low. In image regions with slowly varying intensity (smooth gradients like sky, skin, walls), insufficient gray levels create abrupt, band-like transitions — resembling topographic contour lines that don't actually exist in the original scene.
Cause: Insufficient gray-level quantization. Smooth intensity gradients get "rounded" to the nearest available level, creating artificial steps.
Example: An 8-bit (256-level) image reduced to 3-bit (8 levels) — smooth skin tones show visible stripes.
Q. Explain what you would expect the result of Sampling and Quantization operations on image.
2024 · Part B Q1 · 4 marks ⭐ Full answer in theory section above
Full answer provided in the RTU Numerical box in the theory section above.

Summary:
• Sampling reduces M×N: Loss of spatial resolution → pixelization (blocky appearance), staircase at curved edges, aliasing of high-frequency details. The Nyquist theorem states sampling frequency must be ≥ 2 × max spatial frequency.

• Quantization reduces k (bits): Loss of gray-level resolution → false contouring in smooth gradient regions, fewer tonal distinctions. Below about 50 gray levels, human eye perceives false contours.

Both operations are necessary to digitize a continuous image, but their over-reduction causes specific visible artifacts. The goal is to choose M, N and k values that are sufficient for the application while minimizing storage cost.

Image Acquisition

80% · Part B with Diagram

What is Image Acquisition?

Image acquisition is the process of capturing a physical scene and converting it into a digital image. It involves an illumination source, an object/scene, and an imaging sensor (camera).

Energy from the illumination source hits the object, reflects off it, and is captured by the sensor. The sensor converts the energy into an electrical signal which is then digitized.

Image Acquisition System — Block Diagram (Draw this every time!)

Fig 4.1 — Image acquisition system: Illumination → Object → Lens → Sensor → A/D Converter → Digital Image → Processing

Types of Imaging Sensors

1. Single Sensor (Line Sensor)

A single photodetector. To acquire a 2D image, the sensor must be physically scanned across the scene (mechanically moved row by row). Very slow but high precision. Used in: flatbed scanners, high-precision industrial cameras.

2. Line Sensor (1D Array)

A 1D row of photodetectors. The sensor captures one row of the image at a time. The scene or sensor moves perpendicular to the sensor to capture the full image. Used in: document scanners, fax machines, conveyor-belt inspection.

3. Array Sensor (2D Array) — Most Common

A 2D grid (matrix) of photodetectors that captures the entire image simultaneously. Two main technologies:

  • CCD (Charge-Coupled Device): Better image quality, lower noise. Used in professional cameras, medical imaging, astronomy. Charge transfers sequentially to output.
  • CMOS (Complementary Metal-Oxide-Semiconductor): Cheaper, faster, lower power. Used in smartphones, webcams, consumer cameras. Each pixel has its own amplifier.
PropertyCCDCMOS
Image qualityHigher (less noise)Good but can have noise
Power consumptionHighLow
Speed (frame rate)LowerHigher
CostExpensiveCheaper
On-chip processingLimitedYes (ADC per pixel)
UseMedical, astronomy, DSLRsPhones, webcams, action cams

Illumination Sources

The choice of illumination affects the quality and type of image captured:

  • Visible light: Standard photography. White light for general imaging.
  • Infrared (IR): Night vision cameras, thermal imaging, remote sensing. Objects emit IR radiation proportional to temperature.
  • Ultraviolet (UV): Fluorescence imaging, forensics (detecting substances invisible in visible light).
  • X-rays: Medical imaging (bone, tissue density). Objects attenuate X-rays differently based on density.
  • Gamma rays: Nuclear medicine (PET scans). Very high energy.
  • Radar (microwave): Satellite SAR imaging, weather radar. Penetrates clouds.
  • Ultrasound: Medical ultrasound. Sound waves, not electromagnetic.
🔗
Connection: Acquisition → Unit 4 (Noise)

Different sensors introduce different noise types. CCD sensors in low light produce Gaussian noise (thermal electron fluctuations). Faulty CMOS pixels produce Salt & Pepper noise (stuck pixels). Understanding acquisition helps explain why specific noise models (Unit 4) are used for specific imaging situations.

Q. Explain Image sensing and acquisition with suitable diagram.
June 2022 · Part B Q1 · 10 marks ⭐ Draw the full block diagram!
Image Sensing and Acquisition:

Image acquisition is the process of capturing a scene as a digital image. The acquisition process involves the following components:

[Draw block diagram: Illumination → Scene → Lens/Optics → Image Sensor (CCD/CMOS) → A/D Converter → Digital Image f(x,y)]

1. Illumination Source:
Energy (visible, IR, UV, X-ray, radar) illuminates the scene. The type of illumination determines what properties of objects are captured. Uniform, controlled illumination reduces artifacts.

2. Optical System (Lens):
Focuses reflected/emitted energy from the scene onto the sensor plane. Determines field of view and magnification. Problems: aberration, distortion, defocus blur.

3. Image Sensor:
Converts optical energy (photons) into electrical signals. Two main types:
• CCD: Sequential charge transfer, high quality, low noise, used in medical/scientific imaging
• CMOS: Per-pixel amplification, faster, cheaper, lower power, used in smartphones

Three configurations:
a) Single sensor: scanned mechanically over scene
b) Line sensor (1D array): one row captured at a time
c) Area sensor (2D array): full image captured simultaneously — most common

4. A/D Converter:
Converts analog electrical signal from sensor to digital values.
• Sampling: measures signal at discrete spatial points → creates M×N grid
• Quantization: maps continuous intensity to discrete levels L = 2ᵏ
• Output: digital image f(x,y) stored as M×N matrix

5. Image Processing System:
The digital image is then processed for various applications — enhancement (Unit 3), restoration (Unit 4), compression (Unit 5), segmentation (Unit 6).

Key parameters:
• Spatial resolution: determined by sensor pixel count and optical system
• Gray-level resolution: determined by A/D converter bit depth
• Temporal resolution (video): determined by frame rate
• SNR: signal-to-noise ratio — higher is better quality

Color Image Representation

85% · Part A/B — Links to Unit 3

Why Color?

Human vision uses three types of cone cells (sensitive to red, green, blue wavelengths). Color images carry significantly more information than grayscale — they enable distinguishing objects with similar gray levels but different colors (e.g., red and green objects look identical in grayscale).

⭐⭐ RGB Color Model — Primary Additive

The RGB model is the most fundamental. Every color is created by adding three primary colors — Red, Green, Blue — at varying intensities.

Fig 6.1 — Additive color mixing: R+G+B overlapping circles. Secondary colors at intersections.
RGB Image Structure
Color pixel: P(x,y) = [R(x,y), G(x,y), B(x,y)]
Gray conversion: I = 0.299·R + 0.587·G + 0.114·B (luminance formula)
Green contributes most to perceived brightness (0.587). Blue contributes least (0.114).

⭐ CMY / CMYK Model — Subtractive (Printing)

Printing inks absorb (subtract) light from white paper. Cyan absorbs Red; Magenta absorbs Green; Yellow absorbs Blue.

RGB → CMY Conversion (values in [0,1])
C = 1 − R
M = 1 − G
Y = 1 − B
K (black) is added in CMYK for efficiency: K = min(C, M, Y). Reduces ink usage for dark colors.
C' = (C − K)/(1 − K) [after adding K]

⭐⭐ HSI Model — Human-Perception Based

Separates color information (H, S) from intensity/brightness (I). Directly used in Unit 3. Covered in full detail there — here is the quick summary:

  • H (Hue): The dominant color (0°–360°). Red=0°, Green=120°, Blue=240°.
  • S (Saturation): Purity of color (0=gray, 1=pure color).
  • I (Intensity): Brightness (0=black, 1=white). Completely decoupled from color.
🔗
Unit 2 → Unit 3: Why HSI is Needed

RGB components are correlated with brightness — changing luminosity changes all three R, G, B values together. The HSI model (Unit 3) solves this by separating I (intensity) from H and S (color). This allows operations like histogram equalization to be applied to only the I channel without distorting the color hue and saturation. This is the key motivation taught in Unit 2 that becomes critical in Unit 3.

⭐ Pseudo-Coloring

Pseudo-coloring (or false coloring) assigns colors to a grayscale image for visualization purposes — NOT because the original had those colors. Used to make subtle gray-level differences more visible to human eyes, which can distinguish many more colors than gray levels.

Methods:

  • Intensity Slicing: Map each gray level range to a specific color. Example: levels 0–50 → blue, 51–100 → green, 101–200 → yellow, 201–255 → red.
  • Pseudo-Color LUT (Lookup Table): A predefined mapping table that converts each gray value to (R, G, B). Standard LUTs: rainbow, heat map, jet colormap.
  • Frequency Domain Pseudo-Coloring: Apply different phase shifts to DFT and combine channels — more complex visual encoding.
📌 Pseudo-Coloring Application Example

Medical X-ray: A grayscale X-ray shows bone density from 0–255. Pseudo-coloring maps low density → blue (soft tissue), medium → green (cartilage), high → red/white (dense bone). A doctor can now instantly identify tissue types by color instead of tiny gray differences.

Satellite Thermal Imaging: Temperatures mapped as colors — cold blue → warm orange → hot red. Makes temperature gradients immediately visible.

📝 RTU PYQs — Color Representation
Q. Explain following with example: (a) Lookup Table (b) Additive vs. Subtractive color system (c) Hue and Saturation (d) HMMD Color Space
May 2025 · Part C Q4 · 10 marks ⭐ (also covered in Unit 3)
(a) Lookup Table (LUT):
A lookup table is a precomputed array that maps each possible input gray level (0–255) to a corresponding output value. Instead of computing a transform function pixel-by-pixel, we compute the entire mapping once and store it. Processing speed = O(1) per pixel (just a memory lookup).
Example: Gamma correction with γ=0.5: LUT[r] = 255 × (r/255)^0.5 for r = 0..255. Apply to every pixel: g(x,y) = LUT[f(x,y)].
Used in: contrast adjustment, pseudo-coloring, histogram equalization (Unit 3).

(b) Additive vs. Subtractive Color System:
Additive (RGB): Colors formed by adding light. R+G+B = White. Used in displays/monitors/cameras. Starting from black (no light), add colors to brighten.
Subtractive (CMY): Colors formed by subtracting/absorbing light from white paper using inks. C+M+Y = Black (all absorbed). Used in color printing. Starting from white (paper), add inks to darken.
Relationship: C=1-R, M=1-G, Y=1-B.

(c) Hue and Saturation:
Hue: The perceived dominant color — what makes red different from blue. Measured as an angle (0°–360°) in the HSI color space. Red≈0°, Green≈120°, Blue≈240°.
Saturation: The "purity" or "colorfulness" of the hue. S=0 means pure gray (no color). S=1 means fully saturated (vivid color). Example: Pink = Red with low saturation (white mixed in).

(d) HMMD (Hue-Max-Min-Diff) Color Space:
HMMD is derived from RGB: H=Hue, Max=max(R,G,B), Min=min(R,G,B), Diff=Max-Min.
Diff is similar to Saturation (chroma). This space is used in MPEG-7 color descriptors for image retrieval. It better separates luminance from chrominance than RGB for efficient similarity search in image databases.

Digitization Artifacts

80% · Part A — Very Commonly Asked

⭐⭐⭐ Pixelization

Pixelization occurs when spatial sampling is insufficient — the image does not have enough pixels to represent fine details. Individual pixels become visible as colored squares.

Visual effects of pixelization:

  • Curved edges appear as staircase (jagged) patterns
  • Fine details like text, thin lines, or small objects disappear
  • Image appears "blocky" when displayed at larger size
  • High-frequency content (edges) is aliased as lower-frequency patterns

Prevention: Use higher spatial resolution (more pixels). Anti-aliasing: pre-filter to remove high frequencies before reducing resolution.

⭐⭐⭐ False Contouring (Quantization Contouring)

False contouring (also called quantization contouring or banding) occurs when insufficient gray levels create visible boundaries in what should be smooth tonal transitions.

Why it looks like contours: In a smooth gradient (e.g., sky from light to dark blue), the continuous intensity changes gradually. With only a few gray levels, many adjacent pixels that should have slightly different intensities all get mapped to the same level. At the boundary between levels, there is a sudden jump — creating an artificial "edge" or "contour line."

Most visible in: Smooth gradients, human faces, skies, out-of-focus backgrounds — anywhere intensity changes slowly.

Prevention: Use more bits (higher k). Dithering: add controlled noise before quantization to break up the banding pattern.

⚡
Exam Comparison — 2 marks "Pixelization vs False Contouring"

Pixelization: Due to LOW SAMPLING (not enough pixels). Affects spatial detail. Cure: increase M×N.
False Contouring: Due to LOW QUANTIZATION (not enough bits). Affects tonal smoothness. Cure: increase k (bit depth).
Simple trick to remember: Pixelization = Pixel count (spatial). False contouring = Few levels (intensity).

⭐⭐ Aliasing

Aliasing is the distortion that occurs when the sampling rate is below the Nyquist rate. High-frequency patterns in the original image "masquerade" as lower-frequency patterns in the sampled image.

📌 Classic Aliasing Example

Moiré Pattern: When imaging a fabric with fine diagonal stripes at too low a resolution, the sampling misses the true pattern frequency, and instead a larger-scale moiré pattern (interference pattern) appears in the image — it's an aliasing artifact.

Wagon wheel effect (in video): A wagon wheel spinning fast appears to spin backwards in video — because the sampling rate (frame rate) is below 2× the wheel's rotation frequency.

Solution — Anti-Aliasing: Apply a low-pass filter to the image BEFORE sampling. This removes frequencies above Nyquist (f_max = f_s/2), preventing them from folding back into lower frequencies. This is called pre-filtering or anti-aliasing.

⭐ Quantization Noise

Every quantized pixel has a small error = true value − quantized level. This error is called quantization noise or quantization error.

Quantization Error
e(x,y) = f(x,y) − Q[f(x,y)]
Q[·] = quantization operator. Maximum error = ±Δ/2 where Δ = step size = L_max / (2ᵏ − 1).
With uniform quantization and many levels (large k), e(x,y) is approximately uniformly distributed in [−Δ/2, +Δ/2] — treated as additive uniform noise.

This quantization noise is why even perfect (noiseless) image capture still introduces some error. It's why medical images use 12-bit (4096 levels) rather than 8-bit — to reduce quantization error to clinically acceptable levels.
